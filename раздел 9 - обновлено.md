# 9 Модуль управления оборудованием

Модуль управления оборудованием и бронированием обеспечивает базовый учёт инфраструктуры фитнес-центра в части оборудования.
- Контроль расположения оборудования
- Учёт состояния оборудования
- Техническое обслуживание
- Бронирование оборудования для занятий

## 9.1 Типы оборудования

### 9.1.1 Модель типа оборудования (EquipmentType)
```dart
class EquipmentType {
  String id;
  String name;                // "Гантели", "Беговая дорожка"
  String? description;

  // Классификация
  EquipmentCategory category; // Категория (cardio, strength, etc.)
  
  // Базовые характеристики
  String? weightRange;        // "1-30 кг" (для свободных весов)
  String? dimensions;         // "80x150x200 см"
  bool isMobile;              // Мобильное оборудование
}

enum EquipmentCategory {
  cardio,           // Кардио-оборудование
  strength,         // Силовые тренажёры
  freeWeights,      // Свободные веса
  functional,       // Функциональное оборудование
  accessories,      // Аксессуары
  measurement,      // Измерительное оборудование
  other             // Прочее
}
```

## 9.2 Экземпляры оборудования (EquipmentItem)
Учёт конкретных физических единиц оборудования. Каждый экземпляр имеет инвентарный номер и отслеживается индивидуально.

### 9.2.1 Модель экземпляра оборудования (EquipmentItem)
```dart
class EquipmentItem {
  String id;
  String typeId;              // Ссылка на EquipmentType
  String inventoryNumber;     // Внутренний инвентарный номер "ГАН-001"
  String? serialNumber;       // Заводской серийный номер (опционально)
  String? model;              // Модель "Matrix T7xe"
  String? manufacturer;       // Производитель

  // Локация
  String? roomId;             // Где находится (ссылка на Room)
  String? placementNote;      // "У окна, рядом с колонной"

  // Состояние
  EquipmentStatus status;     // available, inUse, maintenance
  int conditionRating;        // 1-5 (5 = отличное)
  String? conditionNotes;     // "Поцарапана панель, работает нормально"

  // Техническое обслуживание (упрощённое)
  DateTime? lastMaintenanceDate;
  DateTime? nextMaintenanceDate;
  String? maintenanceNotes;   // "Заменили ремень"

  // Покупка/учёт
  DateTime? purchaseDate;
  double? purchasePrice;
  String? supplier;
  int? warrantyMonths;        // Гарантия в месяцах

  // Использование
  int usageHours;             // Накопленные часы использования
  DateTime? lastUsedDate;

  // Фотографии конкретного экземпляра
  List<String> photoUrls;
}
```

### 9.2.2 Статусы оборудования (EquipmentStatus)
```dart
enum EquipmentStatus {
  available,     // Доступно
  inUse,         // Используется (во время занятия)
  reserved,      // Забронировано
  maintenance,   // На обслуживании/ремонте
  outOfOrder,    // Неисправно
  storage,       // На складе
}

// Расширение для получения цвета и названия статуса
extension EquipmentStatusExtension on EquipmentStatus {
  String get title { /* ... */ }
  Color get color {
    switch (this) {
      case EquipmentStatus.available: return Colors.green;
      case EquipmentStatus.inUse: return Colors.blue;
      case EquipmentStatus.reserved: return Colors.orange;
      case EquipmentStatus.maintenance: return Colors.purple;
      case EquipmentStatus.outOfOrder: return Colors.red;
      case EquipmentStatus.storage: return Colors.grey;
    }
  }
}
```

## 9.3 Функционал технического обслуживания оборудования

### 9.3.1 Модель истории ТО (EquipmentMaintenanceHistory)

class EquipmentMaintenanceHistory {
  String id;

  MaintenanceType type; // Профилактическое/Корректирующее

## 9.4 Бизнес-правила и процессы ТО

### 9.4.1 Логика смены статусов заявки на ТО
- **Новая заявка:** Может быть создана только со статусом `reported` (Заявка).
- **Из статуса `reported` (Заявка):** Можно перейти в `inProgress` (В работе) или `cancelled` (Отмена). Нельзя перейти в `completed` (Завершено).
- **Из статуса `inProgress` (В работе):** Можно перейти в `completed` (Завершено) или `cancelled` (Отмена). Нельзя вернуться в `reported` (Заявка).
- **Из статуса `completed` (Завершено) или `cancelled` (Отмена):** Статус изменять нельзя.

### 9.4.2 Логика доступности полей в форме редактирования заявки
- **Блокировка формы:** Форма становится полностью недоступной для редактирования (кроме кнопки "Сохранить"), как только в выпадающем списке выбран финальный статус (`completed` или `cancelled`).
- **Кнопка "Сохранить":** Остается доступной для сохранения финального статуса. Если заявка изначально была загружена в финальном статусе, кнопка "Сохранить" неактивна.
- **Поле "Исполнитель":** Доступно для выбора **только** в статусе `reported`.
- **Поле "Описание выполненных работ":** Доступно для редактирования **только** в статусе `inProgress`.
- **Переход в `inProgress`:** Невозможен, если не выбран "Исполнитель".

## 9.6 Пользовательские интерфейсы

### 9.6.1 Карточка экземпляра оборудования (`EquipmentItemDetailScreen`)
*(изменения во вкладке "История ТО")*

**Таб "История ТО":**
- Отображает отсортированный список всех заявок на ТО для данного экземпляра оборудования (новые заявки вверху).
- Каждая строка в списке (`MaintenanceListTile`) содержит:
    - **Заголовок:** Описание проблемы.
    - **Подзаголовок:** В одну строку (с переносом при необходимости), включает:
        - Номер заявки (например, "№2024-001").
        - Текущий статус заявки, выделенный жирным шрифтом и цветом.
        - Дату создания в формате `дд.мм.гг`.
    - Информацию об архивации (если применимо).
- **Интерактивность:**
    - **Нажатие на строку заявки** открывает карточку детального просмотра (`MaintenanceDetailsScreen`).
    - **Кнопка-меню (три точки)** в конце строки предоставляет действия: "Редактировать", "Архивировать" / "Деархивировать".
- **Кнопка "Создать ТО"** для создания новой заявки.

### 9.6.2 Карточка просмотра заявки на ТО (`MaintenanceDetailsScreen`)
- **Заголовок `AppBar`:** Название оборудования.
- **Меню `AppBar`:**
  - **Кнопка "История статусов":** Открывает экран `MaintenanceStatusHistoryScreen`, где в виде таблицы показана история смены статусов заявки.
- **Основная информация:** Номер заявки, проблема, статус, тип, описание выполненных работ и фото.

### 9.6.3 Форма редактирования заявки на ТО (`EquipmentMaintenanceHistoryEditScreen`)
- **Назначение:** Создание новой или редактирование существующей заявки на ТО.
- **Меню `AppBar`:**
  - **Кнопка "История статусов":** Текстовая кнопка, открывает экран истории смены статусов.
  - **Кнопка "Сохранить":** Сохраняет изменения.
- **Логика доступности полей:** Реализована в соответствии с бизнес-правилами из п. 9.4.2.

### 9.6.4 Экран "История статусов" (`MaintenanceStatusHistoryScreen`)
- **Назначение:** Отображение журнала смены статусов для конкретной заявки.
- **Заголовок `AppBar`:** Содержит номер заявки и название оборудования.
- **Интерфейс:**
  - Таблица без дополнительного заголовка внутри.
  - **Колонки:** "Статус", "Пользователь", "Время".
  - Отображает строки для каждого события: создание заявки, начало работ, завершение, отмена.
