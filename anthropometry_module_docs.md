# Модуль "Антропометрия"

Модуль предназначен для сбора, анализа и визуализации антропометрических данных клиентов. Он позволяет отслеживать динамику физических показателей, получать расчетные данные (индексы, метаболизм) и системные рекомендации.

## 1. Модели данных

Все модели данных находятся в общем пакете `fitman_common`.

### 1.1. Основные модели

*   `AnthropometryMeasurement`: Представляет один срез антропометрических данных, сделанный в определенный момент времени. Включает в себя вес, обхваты, данные биоимпеданса и т.д.
*   `AnthropometryFixed`: Хранит постоянные (или редко изменяемые) данные клиента, такие как рост и обхваты, необходимые для расчета соматотипа.

### 1.2. Расчетные и View-модели

*   `WhtrProfile`: Результат расчета соотношения талии к росту (WHtR), содержит сам коэффициент (`ratio`) и его текстовую оценку (`gradation`).
*   `MetabolicProfile`: Результат расчета метаболизма, содержит базовый метаболизм (`bmr`) и суточный расход калорий с учетом активности (`tdee`).
*   `VisualizationDataPoint`: Специализированная модель для экрана визуализации. Содержит как сырые данные замера (вес, обхваты), так и все расчетные (WHtR, BMR, TDEE) для одной точки во времени. Это позволяет фронтенду за один запрос получить все необходимое для построения графиков.

## 2. Backend

### 2.1. База данных (`database/body.sql`)

*   `anthropometry_measurements`: Таблица для хранения периодических замеров.
*   `anthropometry_fix`: Таблица для хранения постоянных замеров (одна запись на клиента).

### 2.2. API Endpoints

*   `GET /api/admin/clients/<id>/anthropometry`: (Для админов) Получение списка всех замеров клиента.
*   `POST /api/measurements/visualize`: **Новый эндпоинт.** Принимает список ID замеров и возвращает полный набор данных (`List<VisualizationDataPoint>`) для построения всех графиков, включая расчетные.
*   `GET /api/dashboard/client/<userId>`: **Обновленный эндпоинт.** Теперь возвращает данные для дашборда конкретного пользователя, включая список последних 7 замеров (`recentMeasurements`) для построения графика прогресса.
*   Другие эндпоинты для CRUD-операций с замерами, а также для получения расчетных данных (соматотип, WHtR, BMR/TDEE) по одному замеру.

## 3. Frontend

### 3.1. Дашборд клиента (`client_dashboard.dart`)

На главный экран клиента добавлен виджет **"Динамика прогресса"**. Это компактный линейный график, который наглядно показывает тренды по двум ключевым показателям:
*   **Вес (кг)**
*   **Обхват талии (см)**

Данные для графика поступают из обновленного эндпоинта дашборда и не требуют дополнительных запросов.

### 3.2. Список замеров (`anthropometry_list_screen.dart`)

Экран со списком всех замеров клиента был значительно доработан:

*   **Множественный выбор:** Снято ограничение на выбор только двух замеров. Теперь можно выбрать любое количество для анализа.
*   **"Выбрать всё":** Над списком добавлен чекбокс, позволяющий одним нажатием выбрать все доступные (неархивированные) замеры. Чекбокс поддерживает три состояния (выбрано всё / не выбрано ничего / выбрано частично).
*   **Меню "Действия":** Для экономии места на мобильных устройствах ряд кнопок заменен на одну кнопку с иконкой "три точки". При нажатии на нее открывается выпадающее меню со следующими действиями:
    *   `Сравнение` (активно при выборе ровно 2 замеров)
    *   `Анализ` (активно при выборе 1 или 2 замеров)
    *   `Визуализация` (активно при выборе 1 и более замеров)
    *   `Рекомендации` (активно при выборе 1 и более замеров)

### 3.3. Визуализация замеров (`anthropometry_visualization_screen.dart`)

**Новый экран**, который открывается по кнопке "Визуализация". Он использует новый эндпоинт бэкенда для получения всех необходимых данных за один запрос и отображает подробные графики по выбранным замерам:
*   Динамика веса, кг
*   Динамика обхватов, см (плечи, грудь, талия, бедра)
*   Динамика WHtR (соотношение талии к росту)
*   Динамика метаболизма, ккал (BMR и TDEE)
*   Состав тела (Жир, % и Мышцы, кг)

Все графики используют прямые линии для более точного отображения данных без визуальных артефактов.

### 3.4. Система Рекомендаций (`system_recommendation_screen.dart`)

Доступ к экрану осуществляется через меню "Действия". Система использует **детерминированный, основанный на правилах, алгоритм** для генерации персональных рекомендаций. Процесс не использует внешние AI/LLM-сервисы.

**1. Входные параметры для анализа:**

Рекомендации строятся на основе комплексных данных клиента:
*   **Профиль пользователя:** Цель (`goalTrainingId`), Уровень подготовки (`levelTrainingId`), Пол, Возраст.
*   **Постоянные замеры:** Рост, Обхват запястья, Обхват лодыжки.
*   **Данные последнего замера:** Обхваты плеч, талии и бедер.

**2. Процесс генерации:**

Алгоритм выполняет несколько шагов:
1.  **Расчет промежуточных показателей:**
    *   **Соматотип** (Эктоморф, Мезоморф, Эндоморф) — определяется по обхватам запястья и лодыжки.
    *   **Тип фигуры** (Яблоко, Груша, Песочные часы и др.) — определяется по соотношению обхватов плеч, талии и бедер.
    *   **Индекс WHtR** (соотношение талии к росту) и его оценка (Норма, Избыточный вес и т.д.).
2.  **Поиск базовой рекомендации:** На основе **Типа фигуры**, **Цели** и **Уровня подготовки** система находит соответствующий блок текста в таблице `body_shape_recommendations`.
3.  **Поиск уточнения:** На основе оценки **WHtR** и **Цели** система находит дополнительный блок текста в таблице `whtr_refinements`.
4.  **Компоновка и обогащение:**
    *   Итоговый текст собирается путем сложения базовой рекомендации и уточнения.
    *   В начало добавляется объяснение методики расчета (на каких показателях основан вывод).
    *   В конец добавляется справочная информация о соматотипе клиента.

**3. Виды рекомендаций:**

Система генерирует два варианта итогового текста:
*   **Рекомендация для клиента:** Упрощенный текст с основными выводами и советами.
*   **Рекомендация для тренера:** Более детальный и технический вариант для профессионального использования.
