# 8. Подсистема рекомендаций по тренингу (Training Recommendations Subsystem)

Этот модуль предоставляет пользователям рекомендации по тренингу на основе их физических данных и целей. Он использует гибридную модель: ядро с детерминированными алгоритмами и опциональный AI-модуль для расширенных советов.

## 8.1 Модель данных (Data Model)

Система рекомендаций опирается на гибридную модель антропометрии, профиль пользователя и несколько справочников.

**Ключевые сущности:**

*   **`anthropometry_fix`**: Хранит статические или редко изменяемые данные пользователя.
    *   `user_id` (PK, FK на `users`)
    *   `height` (INT): Рост
    *   `wrist_circ` (INT): Обхват запястья
    *   `ankle_circ` (INT): Обхват лодыжки

*   **`anthropometry_measurements`**: Таблица временных рядов, хранящая периодические измерения тела.
    *   `id` (PK)
    *   `user_id` (FK на `users`)
    *   `date_time` (TIMESTAMPTZ): Дата и время замера
    *   `weight` (REAL): Вес
    *   `shoulders_circ`, `breast_circ`, `waist_circ`, `hips_circ` (INT): Обхваты

*   **`types_body_build`**: Справочник правил для расчета соматотипа.

*   **`body_shape_recommendations`**: Справочник базовых рекомендаций на основе типа фигуры, цели и уровня подготовки.

*   **`whtr_refinements`**: Справочник уточнений к рекомендациям на основе индекса WHtR (соотношение талии к росту).

*   **`ai_recommendation_cache`**: Кэширует рекомендации, сгенерированные внешним AI-сервисом.

**Логическое представление моделей (Frontend):**

## 8.2 API Endpoints

Взаимодействие с подсистемой осуществляется через следующие RESTful API эндпоинты:

**Для клиентов:**
*   `GET /api/client/anthropometry/fixed`: Получить постоянные антропометрические данные для аутентифицированного пользователя.
*   `POST /api/client/anthropometry/fixed`: Создать или обновить постоянные антропометрические данные.
*   `GET /api/client/anthropometry`: Получить список всех периодических замеров для аутентифицированного пользователя, отсортированный по дате.
*   `POST /api/client/anthropometry`: Создать или обновить периодический замер.
*   `GET /api/recommendations/<id>`: Сгенерировать и получить комплексную рекомендацию для указанного ID пользователя (основной эндпоинт подсистемы).
*   `GET /api/client/anthropometry/somatotype`: Получить только рассчитанный профиль соматотипа.
*   `GET /api/client/anthropometry/whtr-profiles`: Получить только рассчитанные WHtR-профили.

**Для администраторов:**
*   `GET /api/admin/clients/<id>/anthropometry/fixed`: Получить постоянные данные для любого клиента по ID.
*   `POST /api/admin/clients/<id>/anthropometry/fixed`: Создать/обновить постоянные данные для любого клиента по ID.
*   `GET /api/admin/clients/<id>/anthropometry`: Получить список периодических замеров для любого клиента по ID.
*   `POST /api/admin/clients/<id>/anthropometry`: Создать/обновить периодический замер для любого клиента по ID.


## 8.3 Пользовательские интерфейсы (User Interfaces)

Интерфейс реализован в виде единого экрана с двумя вкладками.

1.  **`new_anthropometry_screen.dart`**: Основной экран, содержащий `TabBar` для переключения между вкладками.

2.  **Вкладка "Постоянные данные" (виджет `fixed_values_view.dart`):**
    *   **Отображение:** Показывает текущие значения `роста`, `обхвата запястья` и `обхвата лодыжки` в полях для редактирования.
    *   **Действие:** Кнопка "Сохранить" позволяет изменить и сохранить эти значения. Кнопка активна только при изменении данных.

3.  **Вкладка "Периодические замеры" (виджет `anthropometry_list_screen.dart`):**
    *   **Отображение:** Прокручиваемый список всех периодических замеров с указанием даты и веса.
    *   **Действия:**
        *   Кнопка "+" (Floating Action Button) для перехода на экран `anthropometry_edit_screen.dart` для добавления нового замера.
        *   Нажатие на элемент списка открывает экран `anthropometry_edit_screen.dart` для редактирования этого замера.
        *   Выбор двух элементов в списке с помощью чекбоксов активирует кнопку "Посмотреть рекомендации".

4.  **`recommendation_screen.dart`**:
    *   **Назначение:** Отображается после нажатия на кнопку "Посмотреть рекомендации".
    *   **Отображение:** Показывает сравнение данных из двух выбранных замеров и сгенерированные для них текстовые рекомендации.

## 8.4 Бизнес-логика (Business Logic)

Основная логика находится в сервисе `RecommendationService` на бэкенде. Процесс выглядит следующим образом:

1.  **Сбор данных (`_getClientData`):** Сервис собирает все необходимые данные по пользователю: его профиль (цель, уровень подготовки), запись из `anthropometry_fix` и последний замер из `anthropometry_measurements`.
2.  **Расчет соматотипа (`_determineSomatotype`):** Вычисляется вероятностный профиль (Эктоморф/Мезоморф/Эндоморф) на основе `wrist_circ`, `ankle_circ` и пола, используя правила из справочника `types_body_build`.
3.  **Определение типа фигуры (`_determineBodyShape`):** Определяется текущий тип фигуры ("Яблоко", "Груша" и т.д.) по соотношению обхватов плеч, талии и бедер из последнего периодического замера.
4.  **Расчет WHtR (`_determineWhtrProfile`):** Вычисляется соотношение талии к росту и соответствующая градация ("Норма", "Избыточный вес" и т.д.), используя рост из постоянных данных и талию из последнего периодического замера.
5.  **Генерация рекомендации (`generateRecommendation`):**
    *   Находится базовая рекомендация в `body_shape_recommendations`, соответствующая типу фигуры, цели и уровню пользователя.
    *   Находится уточняющий текст в `whtr_refinements` на основе градации WHtR и цели.
    *   Комбинируются базовый текст и уточнение.
    *   Добавляется объяснение методологии и текст, относящийся к соматотипу пользователя.
    *   (Опционально) Эти данные могут быть отправлены в AI-модуль для получения более детальных и персонализированных советов.

## 8.5 Архитектура и расположение кода (Architecture and Code Location)

**Бэкенд (`/backend/lib/`):**
```
/modules/clients/
├── controllers/
│   └── anthropometry_controller.dart # API для всех данных антропометрии
├── repositories/
│   └── client_repository.dart    # Содержит всю логику работы с БД для антропометрии
/services/
└── recommendations/
    └── recommendation_service.dart # Ядро бизнес-логики для рекомендаций
```

**Фронтенд (`/frontend/lib/modules/clients/`):**
```
/screens/
├── new_anthropometry_screen.dart   # Главный экран с TabBar
├── anthropometry_list_screen.dart  # Виджет для вкладки "Периодические замеры"
├── anthropometry_edit_screen.dart  # Форма для добавления/редактирования периодического замера
└── recommendation_screen.dart      # Экран для отображения сравнения и рекомендаций
/widgets/
└── fixed_values_view.dart          # Виджет для вкладки "Постоянные данные"
```
