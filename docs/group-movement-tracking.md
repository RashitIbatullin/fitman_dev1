# План реализации модуля отслеживания перемещений участников групп

## 1. Общая идея

Создать систему аудита (исторического лога) для всех перемещений персонала (тренеры, инструкторы, менеджеры) и клиентов между тренировочными группами. Это позволит хранить и анализировать данные о том, кто, когда, куда и на каком основании был перемещен.

### 1.5. Принципы работы

- **Источник правды о текущем состоянии:**
  - **Персонал:** Поля `primary_trainer_id`, `primary_instructor_id`, `responsible_manager_id` в таблице `training_groups` всегда содержат актуальную информацию о текущем персонале группы.
  - **Клиенты:** Таблица `training_group_members` всегда содержит актуальный список клиентов, состоящих в группе.
- **Источник правды об истории:**
  - Новая таблица `group_member_movements` используется исключительно как **неизменяемый архив (лог)**. Она не используется для определения текущего состава, а только для анализа истории и построения отчетов.

## 2. Изменения в базе данных

Будет создана новая таблица `group_member_movements`.

```sql
CREATE TABLE group_member_movements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id),
  user_role VARCHAR(50) NOT NULL, -- 'client', 'trainer', 'instructor', 'manager'
  
  from_group_id UUID REFERENCES training_groups(id), -- Может быть NULL, если это первое вступление
  to_group_id UUID REFERENCES training_groups(id),   -- Может быть NULL, если это выход из последней группы
  
  movement_date TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  reason TEXT, -- Основание для перемещения (свободный текст не менее 5 символов)
  
  moved_by_user_id UUID NOT NULL REFERENCES users(id), -- Кто инициировал перемещение
  
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Добавляем индексы для ускорения поиска
CREATE INDEX idx_group_movements_user ON group_member_movements(user_id);
CREATE INDEX idx_group_movements_from_group ON group_member_movements(from_group_id);
CREATE INDEX idx_group_movements_to_group ON group_member_movements(to_group_id);

-- Убедимся, что одно из полей from/to не NULL
ALTER TABLE group_member_movements 
ADD CONSTRAINT chk_from_to_not_both_null
CHECK (from_group_id IS NOT NULL OR to_group_id IS NOT NULL);
```

## 3. Общий модуль (`fitman_common`)

Учитывая наличие общего модуля, общая модель данных будет размещена в нем.

### 3.1. Модель
- Создать единый файл модели `packages/fitman_common/lib/src/models/group_movement.model.dart`.
- Эта модель будет использоваться как на бэкенде, так и на фронтенде, что позволит избежать дублирования кода. Она будет включать сериализацию/десериализацию JSON с учетом `build_runner`.

## 4. Бэкенд

Изменения будут внесены в модуль `/backend/lib/modules/groups/`.

### 4.1. Модель

- Модель `GroupMovement` будет импортирована из общего пакета `fitman_common`.

### 4.2. Репозиторий (`group_repository.dart`)

- Добавить новые методы в `GroupRepository`:
  - `Future<void> logMovement(GroupMovement movement)`: для записи нового события перемещения.
  - `Future<List<GroupMovement>> getMovementsForUser(String userId)`: для получения истории перемещений пользователя.
  - `Future<List<GroupMovement>> getMovementsForGroup(String groupId)`: для получения истории перемещений по конкретной группе.

### 4.3. Сервис (`group_service.dart`)

Это ключевой компонент, где будет реализована бизнес-логика.

- **Пример (перемещение клиента):** Метод `moveClient(clientId, fromGroupId, toGroupId, reason, movedBy)` должен выполнять следующие действия в рамках одной транзакции:
  1. **Валидация:** Перед выполнением проверить, не превысит ли добавление нового участника лимит `max_participants` в целевой группе `to_group_id`. Если да - вернуть ошибку.
  2. Удалить запись из `training_group_members` для `from_group_id`.
  3. Добавить запись в `training_group_members` для `to_group_id`.
  4. Вызвать `group_repository.logMovement()` для сохранения записи о перемещении.
- **Пример (смена тренера):** Метод `updateGroup()` при обнаружении изменения `primary_trainer_id` должен будет сформировать и сохранить запись о "перемещении" тренера.

### 4.4. Авторизация и права доступа
- **Запись (Создание/Изменение):** Все API эндпоинты, связанные с созданием или изменением перемещений (например, через `moveClient`), должны быть защищены. Доступ на выполнение этих операций должен быть только у ролей `admin` и `manager`.
- **Чтение:** Эндпоинты для чтения (`GET`) должны быть доступны более широкому кругу ролей (`client`, `trainer`, `instructor`, `manager`, `admin`). Однако, на уровне сервиса необходимо реализовать логику, гарантирующую, что пользователь видит только те данные, которые ему положены (например, клиент видит только свои перемещения, тренер - перемещения в своих группах).

## 5. API

Изменения будут внесены в `backend/lib/modules/groups/controllers/training_group_controller.dart`.

- Добавить новые эндпоинты:
  - `GET /api/groups/{id}/movements`: Получить историю перемещений для конкретной группы.
  - `GET /api/users/{id}/movements`: Получить историю перемещений для конкретного пользователя.
- Все эндпоинты, изменяющие данные, должны проверяться на наличие прав доступа (см. п. 4.4).

## 6. Фронтенд

Изменения будут внесены в модуль `/frontend/lib/modules/groups/`.

### 6.1. Модели и Сервисы

- Модель `GroupMovement` будет импортирована из общего пакета `fitman_common`.
- Добавить в `frontend/lib/modules/groups/services/group_service.dart` методы для вызова новых API эндпоинтов.

### 6.2. Пользовательский интерфейс
- **Сценарий перемещения клиента** (доступен для `admin` и `manager`):
  1. В списке участников группы (например, на экране `training_group_edit_screen.dart`) у каждого клиента добавить кнопку/пункт меню "Переместить".
  2. При нажатии открывается диалоговое окно со списком всех доступных для перемещения групп.
  3. После выбора целевой группы и ввода "Основания" (reason), не менее 5 символов, фронтенд отправляет запрос на бэкенд.
  4. Бэкенд выполняет валидацию (см. п. 4.3). В случае ошибки (например, группа заполнена), фронтенд отображает соответствующее уведомление.
  5. После успешного перемещения список участников группы обновляется.
- **Отображение истории:**
  - **Экран деталей группы:** Добавить новую вкладку **"Перемещения"**, где будет отображаться список всех движений участников в этой группе.
  - **Профиль пользователя:** Аналогично, добавить вкладку **"Перемещения"** с историей его нахождения в различных группах.
