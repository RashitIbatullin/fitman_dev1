# 10 Модуль инициализации БД (Seeding Module)

## 10.1 Назначение и цели

Модуль инициализации БД (Seeding) предназначен для автоматического наполнения базы данных тестовыми и демонстрационными данными. Это критически важный компонент для процесса разработки и тестирования, позволяющий быстро разворачивать предсказуемое и консистентное состояние БД для различных сценариев.

**Основные цели модуля:**

*   **Ускорение разработки:** Разработчики могут быстро воссоздать базу данных с необходимым набором данных, не тратя время на ручное наполнение.
*   **Консистентность окружения:** Гарантирует, что все члены команды работают с одинаковым набором базовых данных, что снижает количество ошибок, связанных с расхождением данных.
*   **Тестирование:** Предоставляет различные наборы данных для разных видов тестирования:
    *   Минимальный набор для unit-тестов и быстрой проверки функционала.
    *   Реалистичный набор для разработки и тестирования сложной бизнес-логики (например, составление расписаний).
    *   Большой набор для нагрузочного тестирования.
*   **Демонстрация:** Позволяет быстро развернуть демонстрационную версию приложения с предзаполненными данными для показа заказчику или на презентациях.

## 10.2 Архитектура и расположение кода

Для обеспечения гибкости и масштабируемости, код модуля инициализации вынесен из исполняемого файла и организован по модульному принципу с использованием паттерна "Делегирование".

### 10.2.1 Структура файлов

Вся логика модуля находится в директории `/backend/lib/seeding/`.

```
/backend/
├── bin/
│   └── seed.dart               # Точка входа, парсинг аргументов
└── lib/
    └── seeding/
        ├── database_seeder.dart      # Главный класс-оркестратор
        ├── static_data_seeder.dart   # Класс для создания статических данных (справочников)
        ├── user_seeder.dart          # Класс для создания пользователей и их профилей
        ├── infrastructure_seeder.dart# Класс для создания инфраструктуры (здания, залы)
        ├── group_seeder.dart         # Класс для создания тренировочных групп
        └── recommendation_seeder.dart# Класс для создания данных рекомендательной системы
```

### 10.2.2 Описание компонентов

*   **`seed.dart`**: Исполняемый файл, который является точкой входа. Его единственная задача — разобрать аргументы командной строки (`--demo`, `--medium` и т.д.) и запустить соответствующий метод в `DatabaseSeeder`.
*   **`database_seeder.dart`**: Содержит основной класс `DatabaseSeeder`. Этот класс:
    *   Управляет подключением к базе данных.
    *   Создает экземпляры всех классов-помощников (`UserSeeder`, `GroupSeeder` и т.д.), передавая им объект подключения к БД.
    *   Содержит публичные методы для запуска сценариев (`runDemo()`, `runMediumCenter()`).
    *   Оркестрирует процесс, вызывая методы из классов-помощников в правильной последовательности.
*   **Классы-помощники (`*_seeder.dart`)**: Каждый из этих файлов содержит узкоспециализированный класс, отвечающий за создание определенного типа сущностей (пользователей, групп, оборудования и т.д.). Такой подход упрощает поддержку и добавление новой логики.

## 10.3 Режимы работы (сценарии)

Модуль поддерживает несколько режимов работы, которые можно указать при запуске через флаги командной строки.

*   **`--demo`**: Создает минимальный, предсказуемый набор данных, необходимый для базовой разработки и быстрого запуска приложения.
*   **`--medium`**: Создает расширенный, более реалистичный набор данных, подходящий для разработки и тестирования сложного функционала (например, модуля расписаний). Этот режим является **режимом по умолчанию**.
*   **`--load`**: Создает большое количество пользователей (по умолчанию 100) для проведения нагрузочного тестирования.
*   **`--help`**: Показывает справочную информацию по всем доступным флагам.

## 10.4 Использование

Для инициализации базы данных используется PowerShell-скрипт `init_db_dev.ps1`, который автоматизирует весь процесс: от создания таблиц до наполнения их данными с помощью Dart-сидера.

**Основные команды:**

1.  **Запуск инициализации со средним набором данных (по умолчанию):**
    ```powershell
    .\utils\init_db_dev.ps1
    ```

2.  **Запуск инициализации с минимальным демо-набором данных:**
    ```powershell
    .\utils\init_db_dev.ps1 -Mode demo
    ```

Скрипт автоматически вызовет `dart run backend/bin/seed.dart` с соответствующим флагом (`--medium` или `--demo`).

## 10.5 Бизнес-логика сценариев

### 10.5.1 Сценарий `demo`

Создает минимально необходимый набор сущностей для проверки основных функций:
*   **Пользователи:** по 1 для каждой роли (Администратор, Менеджер, Тренер, Инструктор, Клиент).
*   **Инфраструктура:** 2 здания, несколько залов разного типа.
*   **Связи:** Создаются базовые связи между созданными пользователями (например, менеджер назначается тренеру).
*   **Данные:** Заполняются только самые необходимые данные для запуска.

### 10.5.2 Сценарий `medium`

Создает более обширный и реалистичный набор данных, симулирующий работу небольшого фитнес-центра:
*   **Пользователи:**
    *   1 Администратор
    *   2 Менеджера
    *   5 Тренеров
    *   100 Клиентов со случайными параметрами.
*   **Инфраструктура:** Несколько зданий и залов.
*   **Оборудование:** Несколько единиц оборудования в залах.
*   **Тренировочные группы:**
    *   Создается 3-4 группы разной направленности (например, "Утренняя Йога", "Силовой Пауэрлифтинг", "CrossFit").
    *   Для каждой группы задается расписание на неделю.
    *   Клиенты (часть из 100 созданных) распределяются по этим группам.
*   **Связи:** Создаются связи между персоналом и клиентами/группами.
