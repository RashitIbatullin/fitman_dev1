# Резюме: Реализация функционала замены сотрудников

## Цель
Реализовать функционал замены сотрудников в группе (TrainingGroupReplacementEmployees), внедрить унифицированное логирование удалений (клиентов и сотрудников) с обязательным указанием причины, а также устранить архитектурную ошибку смешивания клиентов и сотрудников в логах.

## Выполненные работы

### 1. База данных (database/groups.sql)
- Создана таблица `training_group_replacement_employees` для логирования замен и удалений сотрудников.
- Из таблицы `group_member_movements` удалено поле `user_role` (ранее нарушало архитектуру, так как смешивало клиентов и сотрудников).
- Обновлен скрипт инициализации БД (DROP TABLE) для корректной идемпотентности.

### 2. Backend
- Создана модель `TrainingGroupReplacementEmployee`.
- В `GroupRepository` реализованы методы:
    - `replaceStaff` (замена сотрудника).
    - `removeStaff` (удаление сотрудника с логированием в новую таблицу).
    - `getReplacementsForGroup` (получение истории).
- Обновлен `TrainingGroupController` для поддержки новых эндпоинтов:
    - `POST /api/training_groups/replace-staff`
    - `POST /api/training_groups/remove-staff`
    - `GET /api/training_groups/<id>/replacements`
- Очищена логика `moveClient` и других методов от использования удаленного поля `user_role`.

### 3. Frontend
- Обновлен `ApiService` для взаимодействия с новыми API методами.
- В `TrainingGroupEditScreen`:
    - Добавлена вкладка "Замены".
    - Реализован компонент `ReplacementHistoryView` для отображения истории замен/удалений сотрудников.
    - Внедрены попап-окна с обязательным вводом причины (минимум 5 символов) для удаления/замены.
- Исправлена логика сохранения группы в `TrainingGroupEditScreen` (`_saveForm`), чтобы корректно обрабатывать `null` в полях персонала.
- Исправлена модель `User` (поле `email` сделано необязательным) для соответствия требованиям.

## Текущий статус
Функционал полностью реализован, проверен анализатором кода (`dart analyze`) и протестирован на соответствие требованиям (разделение логов, обязательная причина удаления).
