7.2 Бизнес-процессы ТО
7.2.1 Создание заявки на ТО
1. Обнаружение проблемы: Сотрудник (тренер, инструктор, администратор) обнаруживает неисправность оборудования
2. Создание заявки:
o Делает фото проблемы с разных ракурсов
o Указывает описание проблемы
o Выбирает оборудование из списка
3. Автоматические действия:
o Оборудование помечается как "на ремонте"
o Отменяются бронирования этого оборудования на период ремонта
o Отправляются уведомления администратору и менеджеру
7.2.2 Назначение исполнителя
1. Выбор исполнителя: Администратор/менеджер выбирает исполнителя из:
o Внутренних сотрудников с правами на ТО
o Вспомогательного персонала (техников)
o Внешних подрядчиков
2. Критерии выбора:
o Уровень компетенции (соответствует сложности работ)
o Специализация (соответствует типу оборудования)
o Занятость (текущая нагрузка)
o График работы (для штатных сотрудников)
7.2.3 Выполнение работ
1. Начало работ: Исполнитель отмечает начало работ
2. Фотофиксация: Делает фото в процессе и после завершения
3. Описание работ: Указывает, что было сделано
4. Завершение: Отмечает завершение работ, указывает когда оборудование снова доступно
7.2.4 Контроль качества
1. Приемка работ: Администратор/менеджер проверяет:
o Качество выполненных работ по фото "после"
o Соответствие описанию работ
o Отсутствие видимых дефектов
2. Активация оборудования: Оборудование возвращается в работу

7.3 Структура таблиц
7.3.1 История обслуживания оборудования (equipment_maintenance_history)
-- История ТО оборудования
CREATE TABLE equipment_maintenance_history (
  id BIGSERIAL PRIMARY KEY,
  equipment_item_id BIGINT NOT NULL REFERENCES equipment_items(id),
  equipment_name VARCHAR(255),
  type SMALLINT NOT NULL,
  status SMALLINT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  started_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ,
  equipment_available_from TIMESTAMPTZ,
  reported_problem TEXT NOT NULL,
  work_description TEXT,
  reported_by BIGINT NOT NULL REFERENCES users(id),
  assigned_to_user_id BIGINT REFERENCES users(id),
  assigned_to_staff_id BIGINT REFERENCES support_staff(id),
  related_booking_id BIGINT REFERENCES equipment_bookings(id),
  caused_downtime BOOLEAN DEFAULT false,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  is_archived BOOLEAN DEFAULT false,
  
  -- Ограничения
  CONSTRAINT assigned_to_check CHECK (
    (assigned_to_user_id IS NULL AND assigned_to_staff_id IS NOT NULL) OR
    (assigned_to_user_id IS NOT NULL AND assigned_to_staff_id IS NULL) OR
    (assigned_to_user_id IS NULL AND assigned_to_staff_id IS NULL)
  ),
  
  -- Индексы
  INDEX idx_maintenance_equipment (equipment_item_id),
  INDEX idx_maintenance_status (status) WHERE NOT is_archived,
  INDEX idx_maintenance_assigned_user (assigned_to_user_id) WHERE assigned_to_user_id IS NOT NULL,
  INDEX idx_maintenance_assigned_staff (assigned_to_staff_id) WHERE assigned_to_staff_id IS NOT NULL
);

7.3.2 Фотографии ТО (maintenance_photos)
-- Фотографии ТО
CREATE TABLE maintenance_photos (
  id BIGSERIAL PRIMARY KEY,
  maintenance_id BIGINT NOT NULL REFERENCES equipment_maintenance_history(id) ON DELETE CASCADE,
  url TEXT NOT NULL,
  comment TEXT,
  timing SMALLINT NOT NULL,
  taken_at TIMESTAMPTZ DEFAULT NOW(),
  taken_by BIGINT REFERENCES users(id),
  INDEX idx_maintenance_photos (maintenance_id, timing)
);

7.5 Пользовательские интерфейсы
7.5.1 Дашборд ТО оборудования
• Общая статистика: Количество активных заявок, оборудования в ремонте
• Списки: Активные заявки, назначенные мне, требующие внимания
• Быстрые действия: Создать заявку, Назначить исполнителя, Завершить работы
7.5.2 Форма создания заявки на ТО
1. Выбор оборудования: Поиск по названию, типу, местоположению
2. Описание проблемы: Текстовое поле + обязательные фото
3. Приоритет: Низкий/Средний/Высокий/Критичный
4. Прикрепление файлов: Фото, видео, документы
7.5.3 Форма назначения исполнителя
1. Фильтры: Уровень компетенции, специализация, занятость
2. Список исполнителей: Внутренние сотрудники + вспомогательный персонал
3. Информация: Текущая нагрузка, график работы, рейтинг
4. Назначение: Сроки выполнения, приоритет, комментарии
7.5.4 Карточка заявки на ТО
• Основная информация: Оборудование, проблема, статус
• Фотогалерея: Фото "до" и "после" в сравнительном режиме
• История действий: Кто, что и когда делал
• Комментарии: Обсуждение проблемы и решений
• Кнопки действий: В зависимости от статуса и прав пользователя

7.6 Интеграция с существующими модулями

• Уведомления о необходимости ТО оборудования
